# Questions Authoring Guide

For every new question file in this folder, start with these two sections:

1. `Explanation`
2. `Interview Thinking Process`

Use this structure:

```ts
/**
 * Explanation:
 * <2-4 lines describing what the solution does and why>
 *
 * Interview Thinking Process:
 * 1) Clarify requirements and edge cases.
 * 2) Pick data structures and approach.
 * 3) Describe algorithm step-by-step.
 * 4) Discuss complexity (time/space).
 * 5) Mention extensions or trade-offs.
 */
```

Recommended style:

- Keep bullets concise and interview-friendly.
- Focus on reasoning before implementation details.
- Tailor points to the specific problem (not generic text).
- Prefer practical trade-offs and edge-case discussion.
